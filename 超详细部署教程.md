- `index.html` （主页面文件）

## 方法一：最简单的方法（推荐，5分钟搞定）

### 第一步：打开 Vercel 网站

1. 打开浏览器（Chrome、Safari、Edge 都可以）
2. 在地址栏输入：**https://vercel.com**
3. 按回车键

### 第二步：注册/登录账号

1. 点击右上角的 **"Sign Up"** 或 **"Log In"** 按钮
2. 选择登录方式（推荐用 GitHub 账号，如果没有就注册一个）：
   - 点击 **"Continue with GitHub"**
   - 如果没有 GitHub 账号：
     * 先访问 https://github.com 注册一个账号（免费）
     * 然后再回到 Vercel 用 GitHub 登录

### 第三步：创建新项目

1. 登录后，你会看到 Vercel 的主页面
2. 点击右上角绿色的 **"Add New..."** 按钮
3. 在下拉菜单中选择 **"Project"**

### 第四步：导入项目

**情况 A：如果你想把代码先放到 GitHub（推荐）**

1. 先打开 https://github.com 并登录
2. 点击右上角的 **"+"** 号，选择 **"New repository"**
3. 填写仓库名称（比如：`resume-analysis`）
4. 选择 **"Public"**（公开）
5. 点击 **"Create repository"**
6. 然后回到你的电脑，打开终端（Terminal），运行以下命令：

```bash
cd /Users/changchun/Desktop/job_scraper
git remote add origin https://github.com/你的GitHub用户名/resume-analysis.git
git branch -M main
git push -u origin main
```

（注意：需要先安装 Git，如果没安装，看下面的"情况 B"）

**情况 B：如果不想用 GitHub（更简单）**

1. 在 Vercel 的 "Import Project" 页面
2. 点击 **"Browse"** 或 **"Upload"** 按钮
3. 直接选择你的 `index.html` 文件上传
4. 或者使用 Vercel 的拖拽功能

### 第五步：配置项目

1. 在 Vercel 的项目配置页面：
   - **Framework Preset**: 选择 **"Other"** 或 **"Static Site"**
   - **Root Directory**: 留空（不填）
   - **Build Command**: 留空（不填）
   - **Output Directory**: 留空（不填）

2. 点击页面底部的 **"Deploy"** 按钮

### 第六步：等待部署

1. 你会看到一个部署进度条
2. 等待 1-2 分钟（通常很快）
3. 看到绿色的 **"Success"** 或 **"Ready"** 就完成了！

### 第七步：获取链接

1. 部署完成后，你会看到一个类似这样的链接：
   ```
   https://your-project-name.vercel.app
   ```
2. 点击这个链接，就能访问你的网站了！
3. 这个链接可以分享给任何人使用

## 方法二：通过命令行部署（如果你熟悉终端）

### 第一步：安装 Node.js

1. 访问 https://nodejs.org
2. 下载并安装（选择 LTS 版本）
3. 安装完成后，打开终端验证：
   ```bash
   node --version
   npm --version
   ```

### 第二步：安装 Vercel CLI

在终端运行：
```bash
npm install -g vercel
```

### 第三步：登录 Vercel

```bash
vercel login
```
会打开浏览器让你登录

### 第四步：部署

```bash
cd /Users/changchun/Desktop/job_scraper
vercel --prod
```

按照提示操作即可！

## 🆘 常见问题解决

### Q1: 提示 "找不到文件"？
**A:** 确保你在正确的文件夹里，运行：
```bash
cd /Users/changchun/Desktop/job_scraper
ls index.html
```
应该能看到 `index.html` 文件

### Q2: 部署后显示空白页面？
**A:** 检查：
1. 确保 `index.html` 在项目根目录
2. 查看 Vercel 的部署日志（在项目页面点击 "Deployments"）

### Q3: API 调用失败？
**A:** 检查 `index.html` 中的 API_KEY 是否正确：
- 打开 `index.html`
- 找到 `const API_KEY = '...'`
- 确保 Key 是正确的

### Q4: 如何更新网站？
**A:** 
- 如果连接了 GitHub：修改代码后推送到 GitHub，Vercel 会自动重新部署
- 如果没连接：在 Vercel 项目页面点击 "Redeploy"
